Framework Overview

ShipKit is a unified framework with 24 Claude Code skills organized in three categories: prod-* (12 skills for product discovery), dev-* (9 skills for technical development), and meta skills (3 skills for enforcement, discussion, and extensibility). All artifacts live in .shipkit/ workspace.

Complete Workflow:

prod-* Discovery → /prod-constitution-builder → dev-* Pipeline → /dev-implement → Shipped Code

Product Discovery Skills (12 prod-* skills)

Product discovery and strategy skills. Define what to build and why.

Sequential Skills (10)

Run in order from strategy to success metrics:

/prod-strategic-thinkingStep 1
Define business strategy, value proposition, and market positioning
/prod-constitution-builderStep 2
Product principles - 6 templates (B2B/B2C × Greenfield/Experimental/Side Project/Existing)
/prod-personasStep 3
Identify and characterize target users with demographics and goals
/prod-jobs-to-be-doneStep 4
Map current state workflows and user pain points
/prod-market-analysisStep 5
Analyze competition using Porter's Five Forces
/prod-brand-guidelinesStep 6
Define visual direction, colors, typography, and brand voice
/prod-interaction-designStep 7
Design future state user journeys and interactions
/prod-user-storiesStep 8
Write requirements with MoSCoW prioritization
/prod-assumptions-and-risksStep 9
Identify strategic risks and assumptions to validate
/prod-success-metricsStep 10
Define KPIs, instrumentation, and measurement strategy

Async Skills (2)

Can interrupt any skill when needed:

/prod-discussionAsync
Conversational trade-off analysis and decision facilitation
/prod-communicatorAsync

Development Skills (9 dev-* skills)

Technical specification and development skills. Define how to build it.

Orchestration (2)

Multi-feature project management:

/dev-roadmapOrchestration

/dev-progressOrchestration
Track completion through roadmap - auto-updated after each merge

Core Pipeline (6)

Specification-driven development workflow:

/dev-constitutionCore
Project standards and technical rules
/dev-specifyCore
Feature specifications from descriptions
/dev-planCore
Implementation plans with architecture decisions
/dev-tasksCore
Executable task breakdown with dependencies
/dev-implementCore
TDD execution with integrated verification (includes TDD, verification, debugging)
/dev-finishCore
Merge workflow with test validation + progress tracking

/dev-systematic-debuggingDebugging

Meta Skills (3)

Quick Reference: Common Workflows

Full Product Development (Greenfield)

/prod-strategic-thinking → /prod-constitution-builder → /prod-personas → /prod-jobs-to-be-done → /prod-market-analysis → /prod-brand-guidelines → /prod-interaction-design → /prod-user-stories → /dev-roadmap → /dev-constitution → /dev-specify → /dev-plan → /dev-tasks → /dev-implement → /dev-finish

Single Feature (Existing Codebase)

/dev-specify → /dev-plan → /dev-tasks → /dev-implement → /dev-finish

Multi-Feature Project

/prod-user-stories → /dev-roadmap → (for each feature in roadmap) → /dev-specify → /dev-plan → /dev-tasks → /dev-implement → /dev-finish → (/dev-progress auto-updates)
