# .gitignore Additions for Shipkit

Add these entries to your project's `.gitignore` file when using Shipkit:

```gitignore
# Shipkit - Update check cache (local, checked once per day)
.shipkit/.update-check

# Shipkit - Update archives (local backups, not project state)
.shipkit-archive/

# Shipkit - Work in progress (optional, depends on your workflow)
.shipkit/specs/wip/
.shipkit/plans/wip/
```

## Why ignore .shipkit-archive/?

**Archives are local safety backups** - Created during `/shipkit-update`:
- Contains full backup of previous installation
- Lets you recover if an update causes issues
- Not project state — purely local safety net

**Each update creates a timestamped subfolder:**
```
.shipkit-archive/
├── 20240115-143000/    # First update
├── 20240220-091500/    # Second update
└── ...
```

**Recommendation:** Always ignore `.shipkit-archive/` — these are local backups, not project history.

## Why ignore specs/wip/ and plans/wip/?

**Optional WIP folders** (not yet implemented, future enhancement):
- Draft specs/plans that aren't ready for active/ folder
- Local experimentation, throw-away work
- Move to active/ when ready to implement

**Recommendation:** Ignore WIP folders if you use them, otherwise omit.
