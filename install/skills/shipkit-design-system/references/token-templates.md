# Token Templates

Two complete templates — CSS custom properties and Tailwind config. The skill detects which format to use based on stack.json or engineering-definition.json.

---

## Three-Layer Token Architecture

| Layer | Purpose | When Created | Used By |
|-------|---------|--------------|---------|
| 1. Primitive | Raw palette values | Tier 0 | Only Layer 2 references these |
| 2. Semantic | Intent-based (color-primary, color-error) | Tier 0 | Components reference these |
| 3. Component | Scoped to specific components | Tier 2+ | Only when extraction justifies it |

**Rule:** Never create Layer 3 tokens speculatively. They appear only when a component is promoted to the design system.

## Tailwind Config Template

For Tailwind-based projects, tokens extend the theme. The same three-layer model applies but is expressed through Tailwind's config structure.

```ts
// tailwind.config.ts — Design System Token Extension
// Generated by /shipkit-design-system
// Tier: Seed (Tier 0)

import type { Config } from 'tailwindcss'

export default {
  theme: {
    extend: {
      colors: {
        // Layer 2: Semantic colors (reference in components as bg-primary, text-error, etc.)
        primary: {
          DEFAULT: '#0ea5e9',   // --color-primary
          hover:   '#0284c7',   // --color-primary-hover
          light:   '#e0f2fe',   // --color-primary-light
        },
        accent:    '#38bdf8',
        surface: {
          DEFAULT: '#fafafa',
          alt:     '#f5f5f5',
        },
        border:    '#e5e5e5',
        error:     '#ef4444',
        success:   '#22c55e',
        warning:   '#f59e0b',
        info:      '#3b82f6',
      },
      fontFamily: {
        heading: ['HEADING_FONT', 'system-ui', 'sans-serif'],
        body:    ['BODY_FONT', 'system-ui', 'sans-serif'],
      },
      fontSize: {
        // Only override if the default Tailwind scale doesn't fit
        // Most projects can use Tailwind's built-in text-xs through text-4xl
      },
      spacing: {
        // Only override if the default Tailwind spacing scale doesn't fit
        // Tailwind's 0.5-96 scale covers most needs
      },
      borderRadius: {
        // Tailwind's default sm/md/lg usually works
        // Override only if brand requires specific values
      },
    },
  },
} satisfies Config
```

---

## Customization Notes

- **HEADING_FONT / BODY_FONT**: Replace with actual font names from the design direction. The skill should recommend specific fonts based on the aesthetic tone.
- **Brand colors**: Replace the blue palette with actual brand colors. If the user provides hex codes in `$ARGUMENTS`, use those.
- **Neutral scale**: The default neutral-50 through neutral-900 works for most projects. Adjust warmth (zinc, slate, stone) based on aesthetic tone.
- **Spacing**: The 4px base unit (0.25rem) is standard. Some dense/data-heavy UIs may prefer a 2px or 8px base.
