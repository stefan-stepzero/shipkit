# Brand Guidelines References

This folder contains extended guidance for defining your product's brand personality and visual direction.

## Files in This Folder

### reference.md
Comprehensive guide covering:
- What brand guidelines are and why they matter
- Key components (personality, voice/tone, visual direction)
- Best practices and common mistakes
- Integration with other product discovery skills
- Quick start questions

### examples.md
Real-world examples across different contexts:
- Developer tools (B2B SaaS)
- Consumer wellness app (B2C)
- Financial planning (B2B/B2C)
- Project management (B2B)
- E-learning platform (B2C/B2B)
- Anti-patterns to avoid
- Testing methods

## How to Use These References

1. **Before starting:** Read reference.md to understand brand frameworks
2. **During creation:** Reference examples.md for inspiration
3. **After drafting:** Use testing methods from examples.md to validate

## Adding Your Own References

You can add additional files to this folder:
- PDFs of brand inspiration (competitor analyses, mood boards)
- Screenshots of brand examples you like/dislike
- Links to brand guideline examples online
- Team discussions about brand direction

Claude will read all files in this folder when running the brand-guidelines skill.

## Quick Tips

- **Be specific, not generic:** "Friendly engineer who explains clearly" > "Innovative and user-focused"
- **Ensure alignment:** Voice, tone, and visuals should support the same personality
- **Reference personas:** Brand should resonate with your target users
- **Test with real content:** Apply guidelines to actual UI to see if it feels right
