Reading the Matrix

  • Read across a row — see which agents can use a given skill
  • Read down a column — see all skills available to a given agent
  • ● Has — the agent's persona includes this skill in its toolkit
  • ◆ Spawns — this skill creates an instance of that agent (e.g., master spawns direction/planning/shipping orchestrators)
  • ✓ Human — human can invoke directly via /shipkit-* slash command
  • ~ Auto — automated infrastructure, not designed for direct human invocation

Agent Groups

  • Orchestrators (4) — Orch-Master, Orch-Direction, Orch-Planning, Orch-Shipping — coordinate and dispatch, never produce artifacts directly
  • Producer Workers (5) — Visionary, Product Owner, Architect, Implementer, Thinking Partner — create artifacts
  • Judgment Workers (3) — Rev-Direction, Rev-Planning, Rev-Shipping — assess coherence and quality

The 7 Shipkit Principles

1. MVP ≠ Low Quality

Core paths bulletproof. Everything else minimal. Auth, payments, security are non-negotiable. Edge cases can wait.

2. Context Over Defaults

Check .shipkit/ before assuming. why.json says "MVP in 2 weeks"? Keep it simple. Don't apply "best practices" blindly.

3. Solo Dev Patterns

Skip team ceremony. No PRs, no code reviews, no abstraction "for future developers". Self-documenting code, minimal overhead.

4. AI-First Code

The next developer is also Claude. One concept per file. Explicit imports. "Why" comments. Flat structures. No magic.

5. Modular by Default

<200 lines per file. Types at boundaries. No circular deps. Feature folders. Enables parallel AI agents.

6. SaaS Defaults

User ID on every table. Supabase RLS. Lemon Squeezy webhooks. Basic analytics from day 1. Multi-tenancy baked in.

7. Shipping > Perfecting

Launch with known limitations. Document them in progress.json. Real feedback beats hypothetical requirements.

💡 The Skill Value Test

A skill is valuable if it: forces human decisions OR creates persistence Claude lacks.

If Claude can do it well without instruction (debug, implement, test) → just do it, no skill needed.

Orchestration Loop

Shipkit is a goal-driven 3-loop system. The master sequentially dispatches Direction, Planning, and Shipping loops. Each loop has its own orchestrator, producer workers, and judgment reviewers.

Master (sequencer)
└→ Direction Loop (why, vision, stage, goals + review-direction)
└→ Planning Loop (discovery, definitions, specs + review-planning)
└→ Shipping Loop (plan, test-cases, team, verify, preflight)
